Projects

MuSync

In the fall semester of 2018, I took a course called CMPT 470 (Web-Based Information Systems). We had to form groups of 4 to 6 students and come up with a project idea for a web application that solves a real-world problem. For our problem, we noticed the inconvenience of sharing and listening to music together with friends remotely. Hence, my group, consisting of 6 students, decided to develop a web applcation that allows users to create and join sessions to listen to synchronized music, and chat online. Users must have a Spotify premium account in order to access all the features.

For this project, we utilized the MVC (Model-View-Controller) architecture, LEMP (Laravel, Nginx, MySQL, PHP) stack, Vue.js, and Bootstrap to create our dynamic website. We interfaced with the Spotify Web API to add user authentication, fetch music data, implement a search bar to search for songs, and to create a music player. We also used the service, Pusher, to broadcast chat messages and synchronize music among users in real-time within a session.

Projects

Bounded-Buffer

In the fall semester of 2017, I took a course called CMPT 300 (Operating Systems). In this course, I had to write a program to solve a variation of the Bounded-Buffer Problem (also known as the Producer-Consumer Problem) with various constraints on resources. To solve this problem, I developed a multithreaded program written in C using pthreads and seamphores. In order to avoid deadlock and race conditions, I applied a mutex to lock the critical sections when a thread accessed any shared resources.
